# 接口文档 B+C 组合导航实施计划

## 目标

提升 RustTool 本地工具站接口文档（ApiDocs）的可读性与交互效率。当接口数量增多时，传统的单列滚动长列表定位效率较低。本项目采用 **B（右侧悬浮目录）+ C（顶部过滤与搜索栏）** 组合方案，提供高效的快速检索和直观的平滑滚动定位体验。

---

## 方案设计

### 1. 右侧悬浮目录 (方案 B)
* **布局升级**：将文档区的主栅格从两列（模块导航、主内容区）扩展为三列：
  * 左侧列：模块选择区（固定 `240px`）
  * 中间列：核心文档区（自适应 `minmax(0, 1fr)`）
  * 右侧列：本页目录 TOC（固定 `200px`）
* **响应式隐藏**：当屏幕宽度小于 `1150px` 时，右侧目录自动隐藏，确保中/小屏下的内容展示不拥挤。
* **双向交互**：
  * **正向定位**：点击右侧目录项，页面通过 `scrollIntoView({ behavior: 'smooth' })` 平滑滚动定位到对应的接口卡片，并触发短暂的呼吸灯边框高亮动画。
  * **反向高亮 (Scrollspy)**：监听中间文档区的滚动状态，采用 `IntersectionObserver` 实时检测当前处于视口中部的接口，并高亮右侧目录中对应的导航项。

### 2. 顶部搜索与过滤 (方案 C)
* **文本模糊检索**：实时过滤接口的路径（`path`）、标题（`title`）与描述（`description`）。
* **方法分类过滤**：提供 `ALL`、`GET`、`POST`、`PUT`、`DELETE` 快速筛选标签（Pills）。
* **同步关联**：顶部过滤发生变化时，中间接口卡片列表与右侧目录（TOC）同步更新，保持状态一致。
* **空状态设计**：在无匹配项时显示虚线框提示，引导用户重新过滤。

## 技术实现

### 1. 前端样式 ([base.css](file:///d:/02_ben_work/02_ai/01_openai/RustTool/frontend/src/styles/base.css))
* **三列布局扩展**：
  ```css
  .api-doc-layout {
    @apply grid grid-cols-[240px_minmax(0,1fr)_200px] items-start gap-4 max-[1150px]:grid-cols-[240px_minmax(0,1fr)] max-[980px]:grid-cols-1;
  }
  ```
* **语义化 Method 颜色**：为 `GET`/`POST`/`PUT`/`DELETE` 方法设计专属背景色：
  * `GET` -> `bg-emerald-700` (经典绿)
  * `POST` -> `bg-blue-700` (科技蓝)
  * `PUT` -> `bg-amber-600` (警告黄)
  * `DELETE` -> `bg-red-600` (危险红)
* **平滑过渡与动画**：为防范打包工具对 `@keyframes` 内 `@apply` 的编译限制，使用原生 CSS 属性编写卡片选中高亮动画：
  ```css
  .api-endpoint-card--target {
    animation: api-highlight-pulse 1.8s ease-out;
  }
  @keyframes api-highlight-pulse {
    0% {
      border-color: #10b981;
      background-color: rgba(236, 253, 245, 0.2);
      box-shadow: 0 0 0 2px rgba(6, 95, 70, 0.1);
    }
    100% {
      border-color: #e7e5e4;
      background-color: rgba(250, 250, 249, 0.5);
      box-shadow: none;
    }
  }
  ```

### 2. 前端组件 ([ApiDocs.vue](file:///d:/02_ben_work/02_ai/01_openai/RustTool/frontend/src/pages/ApiDocs.vue))
* **响应式过滤逻辑**：
  ```typescript
  const filteredEndpoints = computed(() => {
    const query = searchQuery.value.trim().toLowerCase()
    const filter = methodFilter.value
    return selectedModule.value.endpoints.filter((endpoint) => {
      if (filter !== 'ALL' && endpoint.method !== filter) return false
      if (query) {
        return endpoint.path.toLowerCase().includes(query) ||
               endpoint.title.toLowerCase().includes(query) ||
               endpoint.description.toLowerCase().includes(query)
      }
      return true
    })
  })
  ```
* **滚动监听 (Scrollspy) 实现**：
  在 `watch` 监听中，通过 `IntersectionObserver` 实时观察当前被选模块及过滤下的所有 DOM 元素：
  ```typescript
  observer = new IntersectionObserver(
    (entries) => {
      entries.forEach((entry) => {
        if (entry.isIntersecting) {
          activeEndpointId.value = entry.target.id.replace('endpoint-', '')
        }
      })
    },
    {
      rootMargin: '-80px 0px -60% 0px', // 精准判定视口上部到中部的激活卡片
      threshold: 0,
    }
  )
  ```
* **平滑跳转**：点击 TOC 触发 `scrollToEndpoint`：
  ```typescript
  function scrollToEndpoint(id: string) {
    const el = document.getElementById(`endpoint-${id}`)
    if (el) {
      el.classList.remove('api-endpoint-card--target')
      void el.offsetWidth // 强制触发重绘以重新播放动画
      el.classList.add('api-endpoint-card--target')
      el.scrollIntoView({ behavior: 'smooth', block: 'center' })
      activeEndpointId.value = id
    }
  }
  ```
