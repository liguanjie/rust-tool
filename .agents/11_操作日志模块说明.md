# 操作日志模块说明

## 目标

Windows 工作台需要记录用户触发的关键操作，便于排查 Docker、sub2api、Clash Party 和系统命令的执行过程。

## 数据表

表名：`operation_logs`

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | INTEGER | 自增主键 |
| `module` | TEXT | 操作所属模块，例如 Docker、sub2api、Clash Party、系统、工作台 |
| `action` | TEXT | 操作动作，例如 检测、启动 Docker、切换节点、保存配置 |
| `status` | TEXT | 状态：`success`、`failed`、`warn`、`running`、`started`、`cancelled` |
| `message` | TEXT | 面向用户的摘要 |
| `detail` | TEXT | 操作明细，避免写入密码、secret 等敏感信息 |
| `created_at` | TEXT | Unix 秒级时间戳 |

## 保留策略

日志保留 7 天。后端在写入日志和读取日志时都会清理过期记录：

```sql
DELETE FROM operation_logs
WHERE CAST(created_at AS INTEGER) < 当前时间戳 - 7天
```

## 当前记录范围

- 工作台配置保存成功/失败
- Docker 自动侦测、状态检测、启动、停止、重启
- Clash Party 自动侦测、状态检测、启动、退出、管理状态刷新、切换订阅、切换节点
- sub2api 启动、停止、升级、健康检查
- 系统关机命令
- 文件/目录选择成功、取消、失败

## 前端展示

Windows 工作台页面新增「操作日志」模块，默认显示最近 80 条日志。日志数据来自 Tauri 命令：

```text
list_operation_logs(limit?: number)
```

后端最多允许一次读取 300 条。
