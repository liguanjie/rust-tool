# 接口在线调试实施计划

## 目标

为 RustTool 的接口文档引入 **在线调试（Try It Out）** 面板，允许开发者直接在网页上向本地运行的后端服务发送真实的 HTTP 请求，并查看实时的响应结果（包括 HTTP 状态码、响应耗时、JSON 格式化数据等）。

---

## 方案设计

### 1. 跨域通信 (CORS) 策略
* 本地开发模式下，前端运行于 `http://127.0.0.1:5173`，后端运行于 `http://127.0.0.1:8080`。
* 后端已在 `crates/rust_tool_server/src/app.rs` 中预先集成了针对这两个本地源的 CORS 跨域放行配置。
* 因此，调试过程采用原生浏览器 `fetch` API 直连本地后端，不经过任何中转或代理插件。

### 2. 界面与交互流设计
* **按钮入口**：在每个接口卡片的 header 内集成带有 `Play` 图标的 “测试接口” 按钮。
* **独立调试上下文**：在 Vue 组件中以字典形式（`testingStates`）隔离不同接口的请求参数、加载态和响应数据。
* **请求编辑**：
  * 对 `POST` / `PUT` 等需要传入 Body 的接口提供多行编辑文本域。
  * 初始化时自动读取接口定义的 `request` JSON 样例填充编辑器。
  * 发送请求前在前端自动进行 `JSON.parse` 校验，格式有误时中断并显示警告，防止无效网络请求。
* **性能与响应统计**：
  * 利用高精度时间戳差值计算出以毫秒（`ms`）为单位的请求耗时。
  * 将返回数据通过 `JSON.stringify(..., null, 2)` 进行排版美化，并在卡片只读区展示。
  * 根据响应状态码动态匹配状态药丸（`2xx` 绿色，其余黄色或红色）。
