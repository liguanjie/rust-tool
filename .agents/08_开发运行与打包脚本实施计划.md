# 开发运行与打包脚本实施计划

## 模块目标

为 RustTool 提供普通用户也能理解的 Windows 脚本入口，减少记命令成本。

脚本设计原则是：常用操作统一入口，细分脚本隐藏在 `scripts` 目录，用户优先双击或通过中文菜单操作。

已完成能力：

- `start.bat` 作为双击入口。
- `scripts/menu.ps1` 提供中文菜单。
- `rt.bat` 提供命令式入口。
- 支持安装前端依赖。
- 支持启动 Web 开发环境。
- 支持启动 Tauri 桌面开发版。
- 支持单独启动 Rust 后端。
- 支持单独启动 Vue 前端。
- 支持运行测试。
- 支持构建 Web release exe。
- 支持构建 Tauri 桌面版。
- 支持运行 release 服务。
- 支持清理构建产物。


## 实施步骤

### 阶段 1：统一入口

1. 在项目根目录提供 `rt.bat`。
2. 每个命令只转发到 `scripts` 下的具体脚本。
3. 根目录提供 `start.bat`，用于双击打开中文菜单。
4. 菜单使用 PowerShell 实现中文交互。

### 阶段 2：环境脚本

1. `env.bat` 统一设置项目根目录和前端目录。
2. 检查 Node、pnpm、cargo 是否可用。
3. 避免每个脚本重复写路径。

### 阶段 3：开发脚本

1. `dev_frontend.bat` 启动 Vite。
2. `dev_server.bat` 启动 Rust API 服务。
3. `dev_desktop.bat` 启动 Tauri 桌面开发版。
4. `dev_all.bat` 同时启动 Web 开发所需服务。

### 阶段 4：构建脚本

1. `build_frontend.bat` 执行前端类型检查和构建。
2. `build_release.bat` 构建 Web release。
3. `build_desktop.bat` 构建 Tauri 桌面版。
4. `run_release_server.bat` 运行 release 服务。

### 阶段 5：测试与清理

1. `test_all.bat` 运行 Rust 测试与前端测试。
2. `clean.bat` 清理 target、dist 等产物。
3. 菜单中保留“清理构建产物”入口。

## 验收标准

- 双击 `start.bat` 不闪退，出现中文菜单。
- 输入数字可以执行对应操作。
- 用户不需要记 pnpm、cargo、tauri 命令。
- `rt test` 可以一键跑测试。
- `rt build-desktop` 可以构建桌面版。
- 命令执行结束后菜单等待回车，用户能看到结果。
