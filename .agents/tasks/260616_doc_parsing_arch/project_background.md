# 办公文档智能解析系统 - 项目背景

## 1. 核心目标
旨在设计并研发一款**企业级/生产级的“办公文档智能解析与分析系统”**。该系统需要能够接收各种复杂排版的办公文档（如 PDF、Word、Excel、甚至是包含图片的扫描件），将其精准地转换为大模型（LLM）易于理解的结构化数据，并最终完成复杂的智能问答、数据提取或业务分析。

## 2. 业务痛点
在传统的 RAG（检索增强生成）或文档分析链路中，文档解析往往是最大的瓶颈：
* **排版混乱**：单纯提取文本会导致多栏排版错乱、图文阅读顺序颠倒。
* **“表格地狱”**：Excel 或 PDF 中的表格直接转文本会丢失单元格对应关系，导致大模型产生严重幻觉。
* **成本与算力困境**：如果把所有页面截图发给大模型（如 GPT-4o 视觉版）成本极高且极慢；如果部署本地百 G 参数的视觉版面大模型，又对服务器显卡要求极苛刻。

## 3. 技术栈选型思考
为了在**“解析精度、处理速度、算力成本”**三者之间取得最佳平衡，项目拟定了以下组件：
* **MarkItDown v2.1**：基础格式与结构转换。作为文档摄入门户，负责将主流数字办公文档快速转译为大模型最友好的 **Markdown** 格式。
* **PaddleOCR v4 (轻量版)**：非结构化与图像提取。作为本地化视觉提取利器，专门对付扫描件和文档内的图片，以极低的 CPU/内存消耗提供高精度的中英文本提取。
* **Pandas**：结构化数据清洗。作为数据处理中枢，专门承接从文档中剥离出来的表格数据，进行过滤、对齐和结构化清洗。
* **gpt-5.5**：顶层智能大脑。作为系统最高级的推理与决策中心，利用超长上下文分析清洗后的 Markdown 并调用数据工具。
